=================================
Django 1.1 RC 1 リリースノート
=================================

:revision-up-to: 17812 (1.4) 

July 21, 2009

Django 1.1 release candidate にようこそ!

これはDjango 1.1イベントリリースまでの三番目 -- そして多分最後の -- 
preview/development リリースで、現在1.1のリリースはこの release candidate の
約一週間後予定されています。 このリリースは Djangoコードベースの新しい機能の
お試しとテストに興味ある開発者を主なターゲットとしていて、彼らは最終の1.1
リリース前に重大なバグの発見と解決を助けてくれるでしょう。

ということで、このリリースは本番使用を目的とはしていないのでそのような使用
をしないでください。

Django 1.1 RC 1 で新しいこと
=============================

Django コードベースは -- １つの例外はありますが -- 最初の 1.1 ベータリリース
から昨日は凍結されているので、このリリース候補にはただ一つの新機能(以下参照)
だけがあります;このリリース候補までの作業はとくに1.1ベータまでに導入された
新機能に関するバグフィックスに注力してきました。

これらの機能の概要に関しては :doc:`Django 1.1 ベータリリースノート
</releases/1.1-beta-1>` を参照してください。


URL ネームスペース
----------------------------

1.1βリリースではDjangoのadminアプリケーションでの逆URL解決が導入されました。
これは :ref:`名前付けURL <naming-url-patterns>` で公開されています。
残念ながらadmin URLの逆解決を一貫性があって正確に行うことを実現現するのは
極端に難しかったので、この問題を解決するためにDjangoに１つの機能を追加しま
した:URLネームスペースです。

簡単に言うと、逆解決が実行されるときに(潜在的にはネスト可能な)名前付けられた
プレフィックスが変換されるので、同じグループのURLを同じアプリケーションで
DjangoのURLConfに何度もインクルードできるということです。詳細は
:ref:`URL ネームスペースを定義する <topics-http-defining-url-namespaces>` 
を参照してください。

この機能をサポートするために必要とした変更により、 admin URLを逆変換される時
に使われるURLパターン名は1.1ベータリリースから変更されています;この機能を
生かしたアプリケーションを開発しているのであれば、コードを更新して新しい名前
にする必要があります(多くの場合、逆変換される名前の ``admin__`` を ``admin:``
に変更することで十分です)。
adminで使われるURLパターン名の全リストと、これにどのようなネームスペースが
適用されるか、に関しては :ref:`admin URL を逆変換する <admin-reverse-urls>`
に関するドキュメントを参照してください。

Django 1.1 ロードマップ
==========================

このリリース候補の時点では Django 1.1 は機能凍結と"文字列凍結"されています
-- Djangoコードベースにおいて翻訳としてマークされていた全ての文字列が
最終のDjango 1.1リリース用の形式に保たれていると言うことです。
リリースを阻害している重大なバグを今から最終1.1リリースの間までに注目して
行きます。

このようなバグが発見されなければ、Django 1.1 はこのリリース候補からおよそ
１週間後の2009年6月29日頃にリリースされます。


手助けできること
=======================

ハイクオリティな1.1リリースを提供するために皆さんの助けが必要です。
このリリース候補は本番使用を意図したもの *ではない* ものの、安全なテスト環境で
このリリース候補を使ってみて見つかった全てのバグや問題を報告することで Django
チームを助けることができます。 Djangoのチケットとラッカーは現在の問題を検索する
中心的な場所です:

* https://code.djangoproject.com/timeline

見つかった問題に対して既存のチケットが対応していないのであれば新しいチケットを
オープンしてください。

さらに1.1リリースより先のことを含めて Django開発の議論は django-developers 
メーリングリストで毎日行われています :

* http://groups.google.com/group/django-developers

... また ``irc.freenode.net`` での ``#django-dev`` IRCチャネルでも。
Djangoの開発を手伝いたいのであればこれらのディスカッションに自由に参加してくだ
さい。

DjangoのオンラインドキュメントにはDjangoへの寄贈の仕方の参照が含まれています:

* :doc:`Djangoへの寄贈の仕方 </internals/contributing/index>`

どのレベルの寄贈も常に歓迎されて評価されます -- コードの開発、ドキュメントの記述
、単にチケットの優先の決定をするとかでもいいですし、提案されたバグフィックスを
テストするのを手伝うなどもあります。
